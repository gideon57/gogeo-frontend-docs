gogeo前端应用开发介绍

# 基本概念

脚手架由两部分组成：baseApp和appkit

baseApp包含登录、权限、基础配置等平台信息，并暴露UI组件、http、lodash、turf等给appkit使用。 appkit开发业务页面，并以组件的形式暴露出来，baseApp根据权限系统返回的菜单权限，动态分配该组件到相应的路由上。

# appkit配置

# 1、appkit配置包名

此处包名为发布后在nginx的文件夹名称

文件目录：appkit/package.json

{
  "name": "myFirstAppkitName",
}

# 2、appkit暴露页面

文件目录：appkit/vue.config.js

  module.exports =  {
    configureWebpack:{
      plugins:[
        new ModuleFederationPlugin({
          exposes: {
            './appkitPage1': './src/views/appkitPage1/index.vue',
            './appkitPage2': './src/views/appkitPage2/index.vue',
          },
        }),
      ]
    }
  }

# baseApp配置

baseApp无须在代码上配置，只需确认：

# 1、appkit前端服务端口为9500（开发阶段）

# 2、菜单接口返回的应用菜单如下：

该应用中文名为我的应用 ，一级路由为myAppName ，由 2 个子页面组成，二级路由分别为 appkitPage1 和 appkitPage2，其中：

页面 appkitPage1 由包名为 myFirstAppkitName 的appkit提供

页面 appkitPage2 由包名为 mySecAppkitName 的appkit提供

  [
    ...,

    {
      path: '/myAppName',
      name: '我的应用',
      isApp: 1,
      children: [
        {
          path: 'appkitPage1',
          name: 'appkitPage1',
          appKit: 'myFirstAppkitName',
          isApp: '1'
        },
        {
          path: 'appkitPage2',
          name: 'appkitPage2',
          appKit: 'mySecAppkitName',
          isApp: '1'
        }
      ]
    }
  ]

# 路由说明

参数 说明 备注
path: '/myAppName' 应用名称，会作为一级路由 系统内唯一
name: '我的应用', 应用中文名称，会作为首页菜单名称
isApp: 1 该菜单是否为appkit菜单 1是0否
children 应用子页面集合

# 路由children说明

参数 说明 备注
appKit: 'myFirstAppkitName' 该页面由包名为 myFirstAppkitName 的appkit提供 系统内唯一
path: 'appkitPage1' 该页面引用 myFirstAppkitName 暴露的 appkitPage1页面 所属appkit内唯一

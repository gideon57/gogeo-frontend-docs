# appkit暴露自身

文件目录： appkit/vue.config.js

脚手架已配置完成，此处 name 为 appkit。

警告： 须确保 name 的值与 baseApp 引入的名称保持一致

  module.exports =  {
    configureWebpack:{
      plugins:[
        new ModuleFederationPlugin({
          name: 'appkit',
        }),
      ]
    }
  }

# baseApp引入appkit

文件目录：baseApp/src/moduleFederation/remotes/index.js

警告： 示例中两处引入的名称：appkit 与 appkit 配置的 name 值保持一致

module.exports =  {
  appkit: 'appkit@http://localhost:9500/remoteEntry.js',
}

# appkit暴露组件

文件目录： appkit/vue.config.js

appKitPage 为暴露的组件名称， './src/views/appKitPage/index.vue' 为组件入口文件

  module.exports =  {
    configureWebpack:{
      plugins:[
        new ModuleFederationPlugin({
          remotes: {
            exposes: {
              './appKitPage': './src/views/appKitPage/index.vue',
            },
          },
        }),
      ]
    }
  }

# baseApp引入组件

  • 在baseApp中创建路由并将引入的组件分配给路由页面

    文件目录： baseApp/src/router/index.js

    示例代码 component: () => import("appkit/appKitPage"), 中 appkit 为appkit的name， appKitPage为appKit暴露的组件名

    {
  path: "/appKitPage",
  name:'appKitPage',
  component: () => import("appkit/appKitPage"),
  hidden: true,
}

    文件目录： permission/.js
